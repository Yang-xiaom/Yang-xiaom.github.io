有关 AI 的一些面试题

发表于2026-05-19|更新于2026-05-19|抽象八股
|浏览量:

平时你是怎么使用 AI 的？（之前疏忽大意的问题）

    在科研方面，因为平时写英文期刊比较多，所以会使用 chatgpt 来润色英文表达。在项目上，实验室有很多横向项目，比如我曾经做过一些传统的图像处理项目，有些时候相关的技术栈没有接触过，所以会用AI去辅助生成技术方案，用AI去辅助编码。对于一些需要维护的老项目，我会使用AI去进行业务逻辑梳理，生成流程图、架构图等。

    日常开发中我常用 Claude Code（deepseek-v4-pro） 辅助开发，面对业务需求会先和 AI 多轮沟通明确需求、敲定技术方案，理清整体逻辑后再进行代码编写，最后自己校验优化，高效完成业务开发。

什么是 SDD？

  • 先把需求、逻辑、边界条件、接口定义写成一份清晰的 spec（规范 / 说明书）
  • 评审确认 spec 没问题
  • 再让 AI（比如 Claude Code）按 spec 生成代码
  • 最后人审核、测试、优化

了解 skill 吗？

    Agent Skill 是大模型可调用的说明文档，每个 Skill 定义了特定任务的执行规范。Skill 超越传统提示词，形成结构化能力封装，不仅描述“做什么”，还规定“怎么做”，实现行为一致性和输出标准化。（固定提示词模板）

什么是 MCP？

  • 所有工具统一放在 MCP Server

  • MCP Client 自动收集全部工具，生成统一版本工具文档

  • 把统一文档喂给任何大模型

  • 所有大模型只能输出同一种格式的调用意图

  • Client 把统一意图 → 翻译成 MCP 标准协议去调用接口

  • 工具执行完返回数据，Client 再转回模型能看懂的内容

    传统函数调用：需要人为手写函数描述，不同大模型调用格式不统一，换模型就要改代码，耦合度高。

    MCP：模型只输出通用调用意图，格式统一，协议转换全自动完成，工具一次封装全模型通用。
